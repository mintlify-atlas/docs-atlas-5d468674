## Overview

Environment variables allow you to configure application settings that differ between development, staging, and production environments without hardcoding values in your source code.

## Vite Environment Variables

The Examen App uses Vite as its build tool, which has specific conventions for environment variables:

<Note>
  In Vite, environment variables must be prefixed with `VITE_` to be exposed to your client-side code.
</Note>

## Environment Files

Vite automatically loads environment variables from these files:

| File | Purpose | Committed to Git |
|------|---------|------------------|
| `.env` | Shared across all environments | ❌ No |
| `.env.local` | Local overrides (not committed) | ❌ No |
| `.env.development` | Development-specific variables | ✅ Yes |
| `.env.production` | Production-specific variables | ✅ Yes |

<Warning>
  Never commit `.env` or `.env.local` files containing sensitive data. Always add them to `.gitignore`.
</Warning>

    ```bash .env
    VITE_FIREBASE_STORAGE_BUCKET=examen-react-2026.firebasestorage.app
    VITE_FIREBASE_MESSAGING_SENDER_ID=1034837524422
    VITE_FIREBASE_APP_ID=1:1034837524422:web:30cebed3f5e241fc77fcc7

    # Application Configuration
    VITE_APP_NAME=Examen App
    VITE_BASE_PATH=/examen-app/
    ```
  </Step>

  <Step title="Add to .gitignore">
    Ensure your `.gitignore` includes:

    ```bash .gitignore
    # Environment variables
    .env
    .env.local
    .env.*.local
    ```
  </Step>

  <Step title="Update Firebase Configuration">
    Modify `src/firebase.js` to use environment variables:

    ```javascript src/firebase.js
    import { initializeApp } from "firebase/app";
    import { getFirestore } from "firebase/firestore";
    import { getAuth } from "firebase/auth";

    const firebaseConfig = {
      apiKey: import.meta.env.VITE_FIREBASE_API_KEY,
      authDomain: import.meta.env.VITE_FIREBASE_AUTH_DOMAIN,
      projectId: import.meta.env.VITE_FIREBASE_PROJECT_ID,
      storageBucket: import.meta.env.VITE_FIREBASE_STORAGE_BUCKET,
      messagingSenderId: import.meta.env.VITE_FIREBASE_MESSAGING_SENDER_ID,
      appId: import.meta.env.VITE_FIREBASE_APP_ID
    };

    const app = initializeApp(firebaseConfig);
    export const db = getFirestore(app);
    export const auth = getAuth(app);
    ```
  </Step>

  <Step title="Create .env.example">
    Create a template file for other developers:

    ```bash .env.example
    # Firebase Configuration
    VITE_FIREBASE_API_KEY=your_api_key_here
    VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
    VITE_FIREBASE_PROJECT_ID=your_project_id
    VITE_FIREBASE_STORAGE_BUCKET=your_project.firebasestorage.app
    VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
    VITE_FIREBASE_APP_ID=your_app_id

    # Application Configuration
    VITE_APP_NAME=Examen App
    VITE_BASE_PATH=/examen-app/
    ```

    This file can be committed to git as a template.
  </Step>
</Steps>

## Accessing Environment Variables

In your React components and JavaScript files, access environment variables using `import.meta.env`:

```jsx
// Access a specific variable
const apiKey = import.meta.env.VITE_FIREBASE_API_KEY;

// Check the current mode
const isDevelopment = import.meta.env.DEV;
const isProduction = import.meta.env.PROD;

// Base URL (automatically set by Vite)
const baseUrl = import.meta.env.BASE_URL;
```

## Built-in Vite Variables

Vite provides several built-in environment variables:

| Variable | Type | Description |
|----------|------|-------------|
| `import.meta.env.MODE` | `string` | The mode the app is running in (development/production) |
| `import.meta.env.DEV` | `boolean` | Whether the app is running in development |
| `import.meta.env.PROD` | `boolean` | Whether the app is running in production |
| `import.meta.env.BASE_URL` | `string` | The base URL the app is being served from |
| `import.meta.env.SSR` | `boolean` | Whether the app is running in server-side rendering |

## Current Configuration Values

The Examen App is currently configured with these base settings:

### Package Configuration

From `package.json`:

```json
{
  "name": "examen-app",
  "version": "0.0.0",
  "homepage": "https://eric-vc.github.io/examen-app/"
}
```

### Vite Configuration

From `vite.config.js`:

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: '/examen-app/' // Base path for GitHub Pages deployment
})
```

<Warning>
  The `base` path in `vite.config.js` must match your GitHub repository name for proper asset loading on GitHub Pages.
</Warning>

### Router Configuration

From `src/App.jsx`:

```jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter basename="/examen-app">
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/examen/:id" element={<Examen />} />
        <Route path="/admin" element={<PanelProfesor />} />
      </Routes>
    </BrowserRouter>
  );
}
```

<Note>
  The `basename` prop in BrowserRouter should match the `base` in vite.config.js for consistent routing.
</Note>

## Environment-Specific Configurations


### Production Environment

Create `.env.production`:

```bash .env.production
# Production-specific settings
VITE_API_URL=https://eric-vc.github.io/examen-app
VITE_DEBUG=false
VITE_ENABLE_ANALYTICS=true
```

## Type Safety (Optional)

For TypeScript projects, create type definitions for environment variables:

```typescript src/vite-env.d.ts
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_FIREBASE_API_KEY: string
  readonly VITE_FIREBASE_AUTH_DOMAIN: string
  readonly VITE_FIREBASE_PROJECT_ID: string
  readonly VITE_FIREBASE_STORAGE_BUCKET: string
  readonly VITE_FIREBASE_MESSAGING_SENDER_ID: string
  readonly VITE_FIREBASE_APP_ID: string
  readonly VITE_APP_NAME: string
  readonly VITE_BASE_PATH: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Environment variables are undefined">
    - Ensure the variable is prefixed with `VITE_`
    - Restart the dev server after adding new variables
    - Check that the `.env` file is in the project root
  </Accordion>

  <Accordion title="Changes not reflecting">
    Stop and restart the development server. Vite loads environment variables at startup.
  </Accordion>

  <Accordion title="Variables work locally but not in production">
    Ensure your deployment platform is configured with the necessary environment variables. GitHub Pages requires variables to be hardcoded or set during build.
  </Accordion>
</AccordionGroup>

## Deployment Considerations

When deploying to GitHub Pages:

1. Environment variables are evaluated at **build time**, not runtime
2. The build process reads from `.env` and `.env.production`
3. Sensitive values should use GitHub Secrets for CI/CD workflows
4. The `predeploy` script runs the build with production environment
