## Results Overview

After submitting an exam, you receive immediate feedback with your grade and detailed answer breakdown. Results include your score, correct/incorrect answers, and optionally a downloadable PDF certificate.

## Results Screen

### On-Screen Display

When you complete an exam (`fase === 'finalizado'`), you'll see:

<Steps>
  <Step title="Success Message">
    **"¡Examen Enviado!"** confirmation at the top of the screen
  </Step>

  <Step title="Grade Display">
    Your final grade displayed in large text:
    - Font size: 4rem
    - Shows one decimal place (e.g., 8.5)
    - Color: Theme accent color
    - Label: "Calificación Final"
  </Step>

  <Step title="PDF Option">
    If enabled by instructor, a download/print button appears with the text:
    
    **"📄 Descargar/Imprimir PDF"**
  </Step>

  <Step title="Navigation">
    Link to return home: "Volver al Inicio"
  </Step>
</Steps>

### Grade Calculation

Your grade is calculated automatically using this formula:

```javascript
let aciertos = 0;
preguntasAleatorias.forEach(p => { 
  if (respuestas[p.texto] === p.correcta) aciertos++; 
});

const nota = preguntasAleatorias.length > 0 
  ? (aciertos / preguntasAleatorias.length) * 10 
  : 0;
```

**Scale:** 0 to 10
- 10 questions, 10 correct = 10.0
- 10 questions, 8 correct = 8.0
- 10 questions, 5 correct = 5.0

<Note>
  Grades are displayed with one decimal place precision (e.g., 7.5, 8.0, 9.2)
</Note>

## PDF Export

### When PDF is Available

The PDF download option appears only if:
```javascript
examenInfo.permitirImpresion === true
```

If enabled, you'll see:
- A blue button: **"📄 Descargar/Imprimir PDF"**
- Clicking this button triggers: `window.print()`
- Your browser's print dialog opens
- You can save as PDF or print physically

### When PDF is Disabled

If the instructor has disabled PDF exports:

```
"La impresión de resultados está desactivada."
```
<Warning>
  You can still view all results on-screen, but cannot download or print them.
</Warning>

## PDF Content Structure

When you print or download the PDF, it includes:

### 1. Official Header (EncabezadoPDF)

The PDF header displays institutional information:

**Logo and Title:**
- Instituto Tecnológico Superior de Zongolica logo
- Institution name
- Your academic unit

**Exam Information Table:**

| Field | Content |
|-------|--------|
| **Asignatura** | Subject name from exam configuration |
| **Opción** | Exam type: Diagnóstico ( X ) / 1ª oportunidad ( X ) / 2ª oportunidad ( X ) |
| **Unidad** | Unit number from exam |
| **Tema** | Topic/theme from exam |
| **Ingeniería** | Engineering program: "Ingeniería en Sistemas Computacionales" |
| **Docente** | Instructor name |

**Student Information Row:**

| Nombre del Alumno(a) | No. De Control | Fecha de aplicación | Calificación |
|---------------------|----------------|---------------------|-------------|
| Your full name | Your control # | Submission date | **Final grade** |

<Note>
  The grade in the PDF is displayed in bold, 12pt font with 1 decimal place.
</Note>

### 2. Detailed Results Table

After the header, the PDF includes a complete answer breakdown:

**Table Structure:**

```html
<h3>Detalle de Resultados:</h3>
<table>
  <thead>
    <tr>
      <th>#</th>
      <th>Pregunta</th>
      <th>Tu Respuesta</th>
      <th>Correcta</th>
    </tr>
  </thead>
  <tbody>
    <!-- One row per question -->
  </tbody>
</table>
```

**For Each Question:**

| Column | Content | Formatting |
|--------|---------|------------|
| **#** | Question number (1, 2, 3...) | Plain text |
| **Pregunta** | Question text | Plain text |
| **Tu Respuesta** | Your selected answer | **Red + bold** if wrong, normal if correct |
| **Correcta** | The correct answer | Plain text |

**Answer Highlighting:**
```javascript
const esCorrecta = respuestas[p.texto] === p.correcta;
// If incorrect:
color: 'red', fontWeight: 'bold'
```

**Unanswered Questions:**
If you didn't answer a question, the table shows:
```
"(Sin responder)"
```

<Warning>
  Incorrect answers are highlighted in **red and bold** to help you identify areas for improvement.
</Warning>

### 3. Signature Section

At the bottom of the PDF:

```
___________________________
     Firma del Alumno
```

A signature line for official documentation purposes.

## Print Styling

The application uses CSS classes to control what appears in print vs. screen:

### `.no-print` Class

Elements that appear ONLY on screen (not in PDF):
- Success message and grade display
- Download button
- Navigation links
- Exam interface during taking

### `.hoja-examen` Class

Elements that appear ONLY in print/PDF:
- Official header with logo
- Results table
- Signature line

**Implementation:**
```jsx
{/* On-screen only */}
<div className="no-print">
  <h1>¡Examen Enviado!</h1>
  <div style={{ fontSize: '4rem' }}>{calificacion.toFixed(1)}</div>
</div>

{/* Print/PDF only */}
<div className="hoja-examen">
  <EncabezadoPDF {...props} />
  <table>...</table>
</div>
```

<Note>
  When you click "Descargar/Imprimir PDF", only the `.hoja-examen` content is included in the output.
</Note>

## Understanding Your Results

### Grade Interpretation

**Passing Grade:** Typically 7.0 or higher (may vary by institution)

**Grade Ranges:**
- **9.0 - 10.0**: Excellent
- **8.0 - 8.9**: Very Good
- **7.0 - 7.9**: Good (Passing)
- **6.0 - 6.9**: Satisfactory (May be passing, check institutional policy)
- **0.0 - 5.9**: Needs Improvement (Failing)

## Result Storage

### What Gets Saved

Your exam results are permanently stored in the Firestore database:

```javascript
await addDoc(collection(db, "resultados"), { 
  examenId: id,
  examenTitulo: examenInfo.titulo,
  nombre: nombre,
  numControl: numControl,
  unidad: unidad,
  calificacion: nota,
  fecha: new Date().toISOString()
});
```

**Stored Information:**
- Exam identifier and title
- Your name and control number
- Academic unit
- Final grade (numerical)
- Submission timestamp

<Warning>
  **Note:** Individual question responses are NOT stored in the database. Only your final grade is saved.
  
  The detailed answer breakdown is only available immediately after submission. If you need to keep this information, download the PDF when available.
</Warning>

### Instructors Can View

Your instructor has access to:
- Your control number
- Your name
- Academic unit
- Final grade
- Submission date/time
- Number of attempts taken

Instructors can filter and export results for all students.

## Multiple Attempts

### Tracking Attempts

The system tracks how many times you've taken each exam:

```javascript
const q = query(
  collection(db, "resultados"), 
  where("examenId", "==", id), 
  where("numControl", "==", numControl.toUpperCase())
);
const snap = await getDocs(q);
const intentos = snap.size;
```

**Each submission:**
- Creates a new record in the database
- Counts toward your attempt limit
- Is visible to your instructor

### Viewing Multiple Results

If you take an exam multiple times:
- Each attempt is saved separately
- Instructors can see all attempts
- Your highest/latest grade may be used (depends on institutional policy)

<Note>
  **Important:** Once you reach the maximum allowed attempts, you cannot retake the exam. Plan carefully before submitting.
</Note>

## Date and Time Information

### Timestamp Format

Submission dates are stored in ISO 8601 format:
```javascript
fecha: new Date().toISOString()
// Example: "2026-03-10T15:30:45.123Z"
```

### Display Format

On the PDF, dates are displayed in Mexican locale:
```javascript
const fechaStr = fecha 
  ? new Date(fecha).toLocaleDateString('es-MX') 
  : "___/___/___";
// Example: "10/03/2026"
```

## Exam Type Indicators

The PDF header shows which type of exam you took:

**Three Options:**
1. **Diagnóstico** - Diagnostic exam

const isDiag = opcion === 'diagnostico' ? 'X' : '_';

## Privacy and Access

### Your Data

**You can see:**
- Your own exam results immediately after submission
- Detailed answer breakdown (on-screen only)
- Your final grade
- Submission timestamp

**You cannot see:**
- Other students' results
- Historical attempts (only current one on screen)
- Question bank or unused questions

### Instructor Access

Instructors can view:
- All student results for their exams
- Aggregate statistics
- Filtered views by academic unit, control number, etc.
- Export data to CSV

<Note>
  Your individual question responses are not stored long-term. Only your final numerical grade is saved to the database.
</Note>

## Academic Integrity

### Result Authenticity

The PDF includes:
- Official institutional header with logo
- Your control number for verification
- Submission date/time
- Instructor name
- Academic unit

**This makes the PDF suitable for:**
- Official academic records
- Course documentation
- Credit verification

### No Editing

<Warning>
  Results are final once submitted. You cannot edit or retake the exam after submission (unless you have remaining attempts).
  
  Double-check your answers before clicking "Finalizar y Enviar".
</Warning>
