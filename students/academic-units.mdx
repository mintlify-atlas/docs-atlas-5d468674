## Overview

The Examen App serves multiple academic units (campuses) within the Instituto Tecnológico Superior de Zongolica (ITSZ) system. Students must select their academic unit during exam registration for proper record-keeping and result organization.

```javascript
const UNIDADES_ACADEMICAS = [

  "Cuichapa", 
  "Tezonapa", 
  "Tehuipango"
];
```

## Selecting Your Academic Unit

### During Registration

When starting an exam, you'll see a dropdown menu labeled **"Unidad Académica"**:

```jsx
<label>Unidad Académica:</label>
<select value={unidad} onChange={e=>setUnidad(e.target.value)} style={inputStyle}>
  <option value="">-- Selecciona --</option>
  {UNIDADES_ACADEMICAS.map(u=>
    <option key={u} value={u}>{u}</option>
  )}
</select>
```

<Steps>
  <Step title="Open Dropdown">
    Click on the "Unidad Académica" dropdown menu
  </Step>

  <Step title="Select Your Campus">
    Choose your academic unit from the list of 7 options
  </Step>

  <Step title="Verification">
    Ensure you select the correct unit - this cannot be changed after submission
  </Step>
</Steps>

<Warning>
  **Important:** You must select an academic unit before the "Comenzar" (Start) button becomes active. The field is required and cannot be empty.
</Warning>

### Selection Validation

The system validates that you've selected a unit:

```javascript
if (!nombre || !numControl || !unidad || errorControl) 
  return alert("Revisa datos");
```

If you try to start without selecting a unit, you'll see an alert: **"Revisa datos"** (Check your data)

## How Academic Units Are Used

### 1. Result Storage

Your academic unit is stored with your exam results:

```javascript
await addDoc(collection(db, "resultados"), { 
  examenId: id,
  examenTitulo: examenInfo.titulo,
  nombre: nombre,
  numControl: numControl,
  unidad: unidad,  // ← Academic unit stored here
  calificacion: nota,
  fecha: new Date().toISOString()
});
```

**Stored Field:** `unidad`

**Purpose:**
- Identifies which campus the student belongs to
- Enables filtering and reporting by campus
- Helps instructors organize results

### 2. PDF Documentation

Your academic unit appears on the official PDF header:

```jsx
<EncabezadoPDF 
  asignatura={examenInfo.titulo} 
  unidad={examenInfo.unidad}
  tema={examenInfo.tema}
  opcion={examenInfo.opcion}
  alumnoNombre={nombre} 
  numControl={numControl} 
  unidadAcademica={unidad}  // ← Your selected unit
  calificacion={calificacion}
  fecha={new Date().toISOString()} 
/>
```

**Display Location:**
```html
<div style={{ textAlign: 'center', flexGrow: 1 }}>
  <h3>Instituto Tecnológico Superior de Zongolica</h3>
  {unidadAcademica && 
    <p>Unidad Académica: <strong>{unidadAcademica}</strong></p>
  }
</div>
```

<Note>
  Your academic unit appears prominently on the PDF header, right below the institutional name and logo.
</Note>

### 3. Instructor Filtering

Instructors can filter results by academic unit in the admin panel:

```jsx
<div style={{flex:'1 1 120px'}}>
  <label style={{fontSize:'0.8em'}}>Unidad:</label>
  <select 
    value={filtros.unidad} 
    onChange={e => setFiltros({...filtros, unidad: e.target.value})} 
    style={{...inputStyle, width:'100%'}}
  >
    <option value="">Todas</option>
    {UNIDADES_ACADEMICAS.map(u => 
      <option key={u} value={u}>{u}</option>
    )}
  </select>
</div>
```

**Filtering Logic:**
```javascript
const resultadosFiltrados = resultados.filter(r => {
  if (filtros.unidad && r.unidad !== filtros.unidad) return false;
  return true;
});
```

This allows instructors to:
- View results for a specific campus
- Generate campus-specific reports
- Compare performance across units
- Export data filtered by campus

### 4. Statistics and Reporting

Academic units enable:
- **Campus-level analytics**: Average grades per unit
- **Enrollment tracking**: Students per campus
- **Resource planning**: Exam usage by location
- **Comparative analysis**: Performance between campuses

## Data Integrity

### Why Dropdown Selection?

The system uses a dropdown (not free text) because:

✅ **Prevents typos**: "Zongolica" vs "Zongolica" vs "zongolica"
✅ **Ensures consistency**: All records use exact same spelling
✅ **Enables filtering**: Instructors can reliably filter by campus
✅ **Data validation**: Only valid units can be selected
✅ **Database queries**: Exact matches for querying results

## Common Questions

<AccordionGroup>
  <Accordion title="What if my campus isn't listed?">
    The seven academic units listed are the official campuses in the ITSZ system. If you believe your campus should be included:
    
    1. Verify you're enrolled in the ITSZ system
    2. Contact your campus administrator
    3. They can request the addition through the system administrator
    
    **Note:** Only system administrators can modify the academic units list in the source code.
  </Accordion>

  <Accordion title="Can I change my academic unit after submission?">
    **No.** Once you submit an exam, your academic unit is permanently saved with your results.
    
    **Important:** Double-check your selection before starting the exam.
    
    If you submitted with the wrong unit:
    - Contact your instructor immediately
    - Provide your control number and exam details
    - They may be able to correct the record manually
  </Accordion>

  <Accordion title="Does my academic unit affect my grade?">
    **No.** Your academic unit is used only for:
    - Record organization
    - Filtering and reporting
    - Documentation on PDFs
    
    It does not affect:
    - Your exam questions
    - Grading algorithms
    - Maximum attempts
    - Available exams
  </Accordion>

  <Accordion title="Why does the PDF show a different unit than I selected?">
    The PDF header actually shows TWO unit references:
    
    1. **Exam Unit** (`examenInfo.unidad`): The unit/chapter of course material
    2. **Your Academic Unit** (`unidadAcademica`): Your campus
    
    These are different fields:
    - Exam Unit = "Unit 3" (course content)
    - Academic Unit = "Zongolica" (your campus)
    
    Make sure you're looking at the "Unidad Académica" field in the header.
  </Accordion>

  <Accordion title="Can instructors see which unit I'm from?">
    **Yes.** Instructors can:
    - View your academic unit in the results table
    - Filter results to show only students from specific units
    - Export data including academic unit information
    
    This is necessary for:
    - Campus-level reporting
    - Resource allocation
    - Academic planning
    
    Your academic unit is not private information within the institutional system.
  </Accordion>
</AccordionGroup>

  </Step>

  <Step title="Deploy Updates">

</Warning>

### Removing Academic Units

<Note>
  **Caution:** Do not remove academic units that have existing student records in the database. This will cause:
  - Orphaned records (students with units not in dropdown)
  - Filtering errors
  - Inconsistent reporting
  
  Instead, consider:
  - Keeping the unit in the list (even if inactive)
  - Migrating existing student records first
  - Adding a status flag (requires code modification)
</Note>

## Technical Implementation

### Dropdown Rendering

The academic unit dropdown is rendered using array mapping:

```jsx
<select value={unidad} onChange={e=>setUnidad(e.target.value)}>
  <option value="">-- Selecciona --</option>
  {UNIDADES_ACADEMICAS.map(u=>
    <option key={u} value={u}>{u}</option>
  )}
</select>
```

**How it works:**
1. Empty option as placeholder ("-- Selecciona --")
2. Map over `UNIDADES_ACADEMICAS` array
3. Create one `<option>` per unit
4. Use unit name as both `key` and `value`
5. Display unit name as option text

### State Management

```javascript
// Initialize as empty
const [unidad, setUnidad] = useState('');

// Update on selection
setUnidad(e.target.value);

// Validate before submission
if (!unidad) return alert("Revisa datos");

// Store in database
await addDoc(collection(db, "resultados"), { 
  unidad: unidad,
  // ... other fields
});
```

### Query Filtering

Instructors filter results by unit:

```javascript
const resultadosFiltrados = useMemo(() => {
  return resultados.filter(r => {
    // If filter is set and doesn't match, exclude
    if (filtros.unidad && r.unidad !== filtros.unidad) 
      return false;
    
    return true;
  });
}, [resultados, filtros]);
```

**Performance Optimization:**
- Uses `useMemo` to cache filtered results
- Only recalculates when `resultados` or `filtros` change
- Exact string matching (`!==`)
