## Overview

The Examen App supports printing two types of documents:

<CardGroup cols={2}>
  <Card title="Blank Exam (Vacío)" icon="file">
    Complete exam with all questions and answer options, without correct answers marked
  </Card>
  <Card title="Answer Key (Clave)" icon="key">
    Shows only questions with correct answers marked for grading
  </Card>
</CardGroup>

## Print Modes

The print system uses a special print mode that renders a printable view:

```javascript Print Mode State
const [modoImpresion, setModoImpresion] = useState(null);
// Values: null | 'vacio' | 'respuestas'
```

## Printing Process

<Steps>
  <Step title="Select Exam">
    Choose an exam from the dropdown in the **2. Gestionar** section.
    
    <Note>
      Print buttons are only visible when an exam is selected and has questions.
    </Note>
  </Step>
  
  <Step title="Choose Print Type">
    Click one of the print buttons:
    
    ```javascript PanelProfesor.jsx:367-370
    <div style={{display:'flex', gap:'10px', marginBottom:'15px'}}>
      <button onClick={() => handleImprimirAdmin('vacio')}>
        🖨️ Vacío
      </button>
      <button onClick={() => handleImprimirAdmin('respuestas')}>
        🔑 Clave
      </button>
    </div>
    ```
  </Step>
  
  <Step title="Print View Opens">
    The system switches to print mode and triggers the browser print dialog:
    
    ```javascript Print Handler
    const handleImprimirAdmin = (modo) => {
      setModoImpresion(modo);
      setTimeout(() => {
        window.print();
      }, 500); // Delay to ensure render completes
    };
    ```
  </Step>
  
  <Step title="Print or Save as PDF">
    Use your browser's print dialog to:
    - Print to physical printer
    - Save as PDF
    - Adjust print settings (margins, orientation, etc.)
  </Step>
  
  <Step title="Close Print View">
    Click the **❌ Cerrar** button in the top-right to return to normal view:
    
    ```javascript PanelProfesor.jsx:270
    <button onClick={() => setModoImpresion(null)}>❌ Cerrar</button>
    ```
  </Step>
</Steps>

## CSS Print Styles

The app uses CSS classes to control what appears in print:

<Tabs>
  <Tab title=".no-print">
    Elements with this class are hidden during printing:
    
    ```css
    @media print {
      .no-print {
        display: none !important;
      }
    }
    ```
    
    Applied to:
    - Navigation buttons
    - Theme controls
    - Close button in print view
  </Tab>
  
  <Tab title=".hoja-examen">
    Container for printable content:
    
    ```css
    .hoja-examen {
      width: 100%;
      background: white;
      color: black;
      font-family: Arial, sans-serif;
    }
    ```
    
    Ensures consistent appearance across browsers and printers.
  </Tab>
  
  <Tab title="break-inside: avoid">
    Prevents page breaks within questions:
    
    ```javascript
    <div style={{ breakInside: 'avoid' }}>
      {/* Question content */}
    </div>
    ```
    
    Keeps each question intact on one page.
  </Tab>
</Tabs>

## Print Best Practices

<AccordionGroup>
  <Accordion title="Paper Size and Margins">
    Recommended print settings:
    - **Paper**: Letter (8.5" x 11") or A4
    - **Orientation**: Portrait
    - **Margins**: Default (usually 0.5" or 1cm)
    - **Scale**: 100% (avoid shrinking)
  </Accordion>
  
  <Accordion title="Page Breaks">
    The system uses `breakInside: 'avoid'` to prevent questions from splitting across pages. For exams with many questions:
    
    - Questions may start mid-page
    - Long questions might still break if they exceed page height
    - Consider limiting questions per page during exam design
  </Accordion>
  
  <Accordion title="Saving as PDF">
    Instead of physical printing:
    
    1. Click print button
    2. Select "Save as PDF" in print dialog
    3. Name file descriptively: `Matematicas_Parcial1_Vacio.pdf`
    4. Store digitally for future use
  </Accordion>
  
  <Accordion title="Print Multiple Copies">
    To print for entire class:
    
    1. Generate blank exam
    2. Save as PDF first
    3. Print PDF with desired number of copies
    4. This avoids re-rendering for each copy
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Print Dialog Doesn't Open">
    The print dialog is triggered after a 500ms delay:
    
    ```javascript
    setTimeout(() => { window.print(); }, 500);
    ```
    
    If it doesn't appear:
    - Check browser pop-up blocker
    - Try manual print (Ctrl+P / Cmd+P)
    - Verify exam has questions
  </Accordion>
  
  <Accordion title="Close Button Not Visible">
    The close button has a fixed position:
    
    ```javascript
    style={{ position: 'fixed', top: 10, right: 10, zIndex: 9999 }}
    ```
    
    If hidden:
    - Scroll to top of page
    - Press Escape key to cancel print
    - Refresh page to exit print mode
  </Accordion>
  
  <Accordion title="Questions Cut Off">
    If questions are truncated:
    - Check page margins in print settings
    - Reduce margin size
    - Use landscape orientation for wide content
    - Split into multiple exams
  </Accordion>
  
  <Accordion title="Logo Not Appearing">
    The institutional logo must be available:
    
    ```javascript
    import logoInstitucional from './images/LOGOS.png';
    ```
    
    If missing:
    - Verify image file exists
    - Check image path
    - Logo may not print if network restricted
  </Accordion>
</AccordionGroup>

  <Step title="Archive Creation">

    2. Print answer keys for all exams

    4. Store in shared faculty drive
    5. Reuse in future semesters
  </Step>
</Steps>
