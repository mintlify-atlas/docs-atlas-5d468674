## Question Structure

Each question has the following format:

```javascript Question Object
{
  texto: "What is 2 + 2?",
  opciones: ["3", "4", "5", "6"],
  correcta: "4" // Must match one of the opciones values
}
```

<Warning>
  The `correcta` field must exactly match one of the values in the `opciones` array, not an index.
</Warning>

## Viewing Question Bank

All questions for the selected exam appear in a paginated list below the form.

### Pagination Controls

```javascript Pagination Logic
const obtenerPreguntasPaginadas = () => {
  const ex = examenes.find(e => e.id === examenSeleccionado);
  if (!ex || !ex.preguntas) return [];
  const indiceUltimo = paginaActual * preguntasPorPagina;
  return ex.preguntas.slice(
    indiceUltimo - preguntasPorPagina, 
    indiceUltimo
  );
};
```

<Accordion title="Pagination Options">

## Editing Questions

<Steps>
  <Step title="Click Edit Icon">
    Click the ✏️ icon on any question. The form populates with existing values:

    ```javascript Load Question for Editing
      // Convert correcta value back to option key
    ```

</Note>

## Deleting Questions

<Steps>
  <Step title="Click Delete Icon">
    Click the 🗑️ icon on any question.
  </Step>
  
  <Step title="Confirm Deletion">
    A browser confirmation dialog appears: "¿Eliminar?"
  </Step>
  
  <Step title="Question Removed">
    The question is removed from the array:
    
    ```javascript Delete Question
    const eliminarPregunta = async (idx) => {
      if(!confirm("¿Eliminar?")) return;
      const ref = doc(db, "examenes", examenSeleccionado);
      const s = await getDoc(ref);
      const nuevasPreguntas = s.data().preguntas.filter((_, i) => i !== idx);
      await updateDoc(ref, { preguntas: nuevasPreguntas });
      cargarExamenes();
    };
    ```
  </Step>
</Steps>

<Warning>
  Deleting questions is permanent. There is no undo functionality. Consider exporting questions to CSV before making bulk deletions.
</Warning>

    ```javascript PanelProfesor.jsx:391-396
    <input placeholder="Opción 4" value={pregunta.op4} required />
    ```
  </Tab>
  
  <Tab title="Data Integrity">
    The correct answer is stored as the actual option text, not an index:
    
    ```javascript
    // ❌ Wrong
    { correcta: 0 }
    
    // ✅ Correct
    { correcta: "The actual answer text" }
    ```
    
    This ensures students see the correct answer even if options are shuffled.
  </Tab>
</Tabs>

## Best Practices

<AccordionGroup>
  <Accordion title="Question Writing Tips">
    - Keep question text concise and clear
    - Avoid "all of the above" or "none of the above" options
    - Make all options plausible to avoid obvious answers
    - Test questions have exactly 4 options (no more, no fewer)
  </Accordion>
  
  <Accordion title="Organization">
    - Use consistent formatting for similar question types
    - Group related questions by topic using the pagination
    - Review questions before publishing exams to students
  </Accordion>
  
  <Accordion title="Editing Strategy">
    - Edit questions immediately after creation to catch typos
    - Avoid editing questions after students have taken the exam
    - Consider creating new exam versions instead of modifying active exams
  </Accordion>
</AccordionGroup>
