## Accessing Results

## Results Data Structure

Each result record contains:


## Loading Results

```javascript Load Results Query
const cargarResultados = async () => {
  setCargandoResultados(true);
  try {
    const resQ = query(
      collection(db, "resultados"), 
      orderBy("fecha", "desc") // Most recent first
    );
    const resSnap = await getDocs(resQ);
    setResultados(resSnap.docs.map(d => ({ id: d.id, ...d.data() })));
  } catch (error) { 
    console.error(error); 
  } finally { 
    setCargandoResultados(false); 
  }
};
```

<Info>
  Results are ordered by date (newest first) using Firestore's `orderBy` clause.
</Info>

### Filter Logic

Filters are applied using `useMemo` for performance:

```javascript Filter Implementation
const resultadosFiltrados = useMemo(() => {
  return resultados.filter(r => {
    // Filter by exam
    if (filtros.examenId && r.examenId !== filtros.examenId) {
      return false;
    }
    
    // Filter by control number (partial match)
    if (filtros.numControl && 
        !r.numControl.includes(filtros.numControl.toUpperCase())) {
      return false;
    }
    
    // Filter by academic unit
    if (filtros.unidad && r.unidad !== filtros.unidad) {
      return false;
    }
    
    return true; // Passes all filters
  });
}, [resultados, filtros]);
```

<Tip>
  The `useMemo` hook ensures filters only recalculate when `resultados` or `filtros` change, improving performance with large datasets.
</Tip>

## Exporting Results

Export filtered results to CSV for external analysis or record-keeping.

<Steps>
  <Step title="Apply Filters">
    Use the filter options to select specific results (or leave filters empty for all results).
  </Step>
  
  <Step title="Click Export Button">
    Click **📥 Reporte CSV** in the top-right corner:
    
    ```javascript PanelProfesor.jsx:448
    <button onClick={handleExportarResultados}>
      📥 Reporte CSV
    </button>
    ```
  </Step>
  
  <Step title="File Downloaded">
    A file named `Reporte.csv` is downloaded with your filtered results.
  </Step>
</Steps>

### Export Format

```javascript Export Implementation
const handleExportarResultados = () => {
  if (resultadosFiltrados.length === 0) {
    return alert("Sin datos");
  }
  
  let csvContent = "data:text/csv;charset=utf-8,Control,Nombre,Unidad,Examen,Calificacion,Fecha\n";
  
  resultadosFiltrados.forEach(r => {
    const clean = (t) => t ? `"${t.toString().replace(/"/g, '""')}"` : "";
    csvContent += [
      clean(r.numControl), 
      clean(r.nombre), 
      clean(r.unidad), 
      clean(r.examenTitulo), 
      r.calificacion.toFixed(2), 
      new Date(r.fecha).toLocaleDateString()
    ].join(",") + "\n";
  });
  
  const link = document.createElement("a");
  link.href = encodeURI(csvContent);
  link.download = "Reporte.csv";
  document.body.appendChild(link);
  link.click();
  document.body.removeChild(link);
};
```

## Deleting Results

<Warning>
  The current implementation does not include a delete function for individual results in the UI. This is intentional to maintain academic integrity and prevent accidental data loss.
</Warning>

If you need to remove a result, use the Firebase Console:

```javascript Delete Function (Not Exposed in UI)
const eliminarResultado = async (id) => {
  if(!confirm("¿Eliminar?")) return;
  await deleteDoc(doc(db, "resultados", id));
  cargarResultados();
};
```

## Performance Considerations

<AccordionGroup>
  <Accordion title="On-Demand Loading">
    Results are only loaded when you click the **Resultados** tab:
    
    ```javascript
    setCargandoResultados(true);
    // ... load data ...
    setCargandoResultados(false);
    ```
    
    This prevents unnecessary database queries when teachers are working in other sections.
  </Accordion>
  
  <Accordion title="Memoized Filtering">
    Filters use `useMemo` to avoid recalculating on every render:
    
    ```javascript
    const resultadosFiltrados = useMemo(() => {
      return resultados.filter(/* ... */);
    }, [resultados, filtros]);
    ```
    
    This is especially important for large result sets (100+ records).
  </Accordion>
  
  <Accordion title="Indexed Queries">
    The results query uses Firestore's `orderBy` which requires an index:
    
    ```javascript
    query(collection(db, "resultados"), orderBy("fecha", "desc"))
    ```
    
    Make sure this index exists in your Firebase Console.
  </Accordion>
</AccordionGroup>
