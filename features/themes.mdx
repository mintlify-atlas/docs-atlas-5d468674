## Overview

The Examen App features a comprehensive theming system with three built-in themes that apply consistently across all pages. Theme preferences are persisted in `localStorage` for a seamless user experience.

## Theme Properties

Each theme defines the following properties:

<AccordionGroup>
  <Accordion title="Core Colors">
    - **bg**: Main background color
    - **text**: Primary text color
    - **card**: Card/container background
    - **border**: Border colors for elements
  </Accordion>
  
  <Accordion title="Interactive Elements">
    - **inputBg**: Input field backgrounds (Examen & PanelProfesor only)
    - **accent**: Accent color for buttons and highlights (PanelProfesor only)
  </Accordion>
</AccordionGroup>

## Implementation

### State Management

Theme state is managed with React hooks and synchronized with localStorage:

```javascript
const [temaActual, setTemaActual] = useState(
  () => localStorage.getItem('temaApp') || 'claro'
);
const styles = THEMES[temaActual];

useEffect(() => {
  localStorage.setItem('temaApp', temaActual);
}, [temaActual]);
```

### Theme Switcher UI

Theme selection is implemented with emoji buttons:


Themes are applied through inline styles using the `styles` object:

```javascript

    borderRadius: '8px'
  }}>
```


## Persistence Behavior

<Steps>
  <Step title="Initial Load">
    On first visit, the app defaults to `'claro'` theme
  </Step>
  
  <Step title="User Selection">
    When user clicks a theme button, the preference is immediately saved to localStorage
  </Step>
  
  <Step title="Cross-Page Consistency">
    All pages read from the same localStorage key (`'temaApp'`)
  </Step>
  
  <Step title="Subsequent Visits">
    On return visits, the saved theme is automatically loaded
  </Step>
</Steps>

## Theme Consistency

<Warning>
While themes are defined separately in each component, the color values are **not exactly identical** across all pages. For example:

- Home uses `card: '#f0f0f0'` for light theme
- Examen uses `card: '#f8f9fa'` for light theme  
- PanelProfesor uses `card: '#f9f9f9'` for light theme

Consider centralizing theme definitions in a shared constants file for perfect consistency.
</Warning>

## Example: Complete Component Integration

```javascript
import { useState, useEffect } from 'react';

const THEMES = {
  claro: { bg: '#ffffff', text: '#333333', card: '#f0f0f0' },
  oscuro: { bg: '#121212', text: '#e0e0e0', card: '#1e1e1e' },
  calido: { bg: '#f5e6d3', text: '#4a3b2a', card: '#e8dcc5' }
};

export default function MyComponent() {
  const [temaActual, setTemaActual] = useState(
    () => localStorage.getItem('temaApp') || 'claro'
  );
  const styles = THEMES[temaActual];

  useEffect(() => {
    localStorage.setItem('temaApp', temaActual);
  }, [temaActual]);

  return (
    <div style={{ 
      minHeight: '100vh', 
      backgroundColor: styles.bg, 
      color: styles.text 
    }}>
      {/* Theme switcher */}
      <div>
        <button onClick={() => setTemaActual('claro')}>🌞</button>
        <button onClick={() => setTemaActual('oscuro')}>🌙</button>
        <button onClick={() => setTemaActual('calido')}>☕</button>
      </div>
      
      {/* Content */}
      <div style={{ background: styles.card }}>
        Your content here
      </div>
    </div>
  );
}
```
